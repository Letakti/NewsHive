# NewsHive

**NewsHive** — Telegram-бот на Python, который собирает новости из RSS-лент и отправляет их пользователю в удобном формате.

## Что умеет бот

- Получать новости по выбранному источнику.
- Получать новости по категориям.
- Отправлять случайные новости.
- Показывать подборку основных (топ) новостей.
- Управлять пользовательскими RSS-источниками (добавление/удаление/просмотр).

## Технологии

Проект реализован на Python и использует:

- `aiogram` — Telegram Bot API и обработка команд/сообщений.
- `aiohttp` — асинхронные HTTP-запросы.
- `feedparser` — парсинг RSS-лент.
- `requests` — вспомогательные HTTP-запросы.
- `aiosqlite` — хранение пользовательских данных в SQLite.

## Запуск проекта

### 1) Установите зависимости

```bash
pip install -r requirements.txt
```

### 2) Настройте конфигурацию

Минимально нужно указать токен бота в переменной окружения:

```bash
export TOKEN='ваш_токен_бота'
```

База данных настраивается через переменные окружения:

- `DATABASE_URL` — основной параметр подключения (поддерживается SQLite URL, например `sqlite:///newshive.db`).
- `SQLITE_PATH` — fallback-путь к файлу SQLite, если `DATABASE_URL` не задан.
- `DB_POOL_SIZE` — ограничение на количество одновременных DB-операций.
- `DB_CONNECT_TIMEOUT_SECONDS` — таймаут подключения к SQLite.
- `DB_BUSY_TIMEOUT_MS` — таймаут ожидания освобождения блокировок SQLite.

Также при необходимости настройте:

- `CHAT_ID` — ваш Telegram ID для личных тестов.
- `NEWS_SOURCES` — словарь RSS-источников.
- `NEWS_CATEGORIES` — категории и привязку к источникам.
- `AUTO_UPDATE_INTERVAL` — интервал автообновления.

### 3) Инициализируйте схему базы данных

Перед первым запуском создайте таблицы:

```bash
python - <<'PY'
import asyncio
from storage.db import init_db

asyncio.run(init_db())
PY
```

### 4) Запустите бота

```bash
python bot.py
```

## Миграция файлового хранилища в БД (перед продом)

Перед переходом в прод перенесите существующие данные из файлов в SQLite:

```bash
python scripts/migrate_file_storage_to_db.py
```

Что делает скрипт:
- читает `user_sources.json`, `user_preferences.json`, `groups.txt`;
- загружает данные в таблицы `user_sources`, `user_preferences`, `bot_groups`;
- работает идемпотентно (повторный запуск не дублирует записи);
- выводит в лог количество мигрированных/обновлённых записей и список пропусков из-за ошибок валидации.
## Миграция существующих данных

Если вы обновляетесь с версии, где данные хранились в JSON/TXT, перенесите данные в SQLite из следующих файлов:

- `user_sources.json`
- `user_preferences.json`
- `groups.txt`

После переноса эти файлы больше не используются рантаймом и могут быть оставлены только как резервная копия.

## Команды и меню

### Telegram-команды

- `/start` — запуск бота и показ главного меню.
- `/help` — список доступных команд.
- `/sources` — открыть меню управления источниками.
- `/news` — получить случайные новости.

Также в коде регистрируется команда `/top` (топ-новости), но основной сценарий получения топ-новостей сейчас реализован через кнопку меню **«📌 Основные новости»**.

### Кнопки главного меню

- **📰 Новости по источнику** — выбор источника и выдача свежих новостей.
- **📂 Новости по категории** — выбор категории и выдача новостей.
- **🎲 Рандомные новости** — случайная подборка новостей.
- **📌 Основные новости** — главные новости из ключевых источников.
- **⚙️ Управление источниками** — переход в меню управления пользовательскими RSS.

### Меню управления источниками

- **➕ Добавить источник** — добавить свой RSS-источник (название + ссылка).
- **➖ Удалить источник** — удалить ранее добавленный источник.
- **📋 Мои источники** — показать список пользовательских источников.
- **🔙 Назад** — вернуться в предыдущее меню.

## Структура (кратко)

- `bot.py` — точка входа, запуск polling и регистрация команд.
- `handlers.py` — обработчики команд и кнопок.
- `parser.py` — работа с RSS и формирование новостных выдач.
- `keyboards.py` — клавиатуры и кнопки меню.
- `config.py` — настройки бота и источников.
- `storage/db.py` — инициализация схемы и операции чтения/записи в БД.

## Примечание по хранению данных

Проект отказался от JSON/TXT-хранилищ (`user_sources.json`, `user_preferences.json`, `groups.txt`) в пользу SQLite для большей консистентности и безопасной конкурентной записи.
